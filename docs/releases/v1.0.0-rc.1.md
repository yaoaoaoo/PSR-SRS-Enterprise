# Release Notes — PSR-SRS Enterprise v1.0.0-rc.1

## Overview

PSR-SRS Enterprise is a locally-runnable, full-stack personalized search ranking
and semantic retrieval system. This Release Candidate includes a complete
FastAPI backend, React TypeScript frontend, SQLite database, and a behavior
feedback loop that powers personalized re-ranking.

## What's Included

### Search & Retrieval
- BM25 keyword search (Okapi weighting, deterministic tokenizer)
- LSA semantic retrieval (TF-IDF + TruncatedSVD, 64-d)
- RRF and Linear hybrid fusion
- Personalized re-ranking with category/brand/price affinity

### Behavior Feedback (E6)
- Impression, click, favorite, add-to-cart, purchase event capture
- Idempotent event submission with client_event_id
- Activity dashboard with event counts and conversion rates
- Event stats API with time-range filtering

### Profile System (E7)
- User profile generation from behavior events
- Base + behavior profile merging
- Single-user and batch profile refresh
- Profile Impact evaluation (base vs personalized ranking comparison)
- Deterministic rebuild from database events

### API
- 13 REST API routes under `/api/v1`
- Unified DataResponse/ErrorResponse/LisatResponse envelopes
- X-Request-ID middleware
- OpenAPI 3.x at `/api/v1/openapi.json`
- Swagger UI at `/docs`

### Frontend
- Search page with all 4 modes + personalization
- Items list and detail
- Users list, detail, and profile view with refresh
- Evaluation with query evaluation and profile impact
- System status with health, readiness, index, and profile management
- Activity page with event stats and recent events
- Responsive layout, loading/error/empty states

## Quick Start

```powershell

# 1. Bootstrap
.venv\Scripts\python.exe scripts\bootstrap_local.py

# 2. Start
.venv\Scripts\python.exe scripts\run_local.py

# Open http://127.0.0.1:8000/docs (API) or http://127.0.0.1:5173 (Frontend)
```

## Known Limitations

- SQLite database for local demo (not production-scale)
- Profiles are in-memory snapshots rebuilt from DB events on restart
- Batch profile refresh operates synchronously
- No time-decay for event weights (recency_decay_enabled=false)
- Profile refresh API is a management operation for trusted local environments only
- No authentication system

## Architecture

- Backend: Python 3.12, FastAPI, SQLAlchemy 2.x, scikit-learn
- Frontend: React 19, TypeScript, Vite 8, React Router 7
- Default database: SQLite with Alembic migrations
